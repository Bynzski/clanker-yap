# Security Policy

## Reporting a Vulnerability

Please do not open public issues for security-sensitive bugs.

Instead, report vulnerabilities privately to the maintainers with:

- a clear description of the issue
- affected versions or commits
- reproduction steps
- impact assessment if known

## Scope

Security-relevant issues include:

- arbitrary command execution
- unsafe file writes
- credential or token exposure
- clipboard or paste behavior that can be exploited unexpectedly
- unsafe model download or update behavior

## Response

The project will aim to:

- acknowledge the report
- reproduce the issue
- prepare a fix
- disclose the issue publicly after a patch is available, when appropriate
